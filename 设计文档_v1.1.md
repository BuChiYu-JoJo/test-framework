## 一、项目背景与目标

### 1.1 背景

AI时代开发周期缩短，测试需要：
- 用例编写门槛低（不懂代码也能参与）
- 新项目适配快（配置驱动，代码复用）
- 复杂场景覆盖（不牺牲测试深度）

### 1.2 目标

1. **降低门槛** — 非技术人员可通过Excel/Web界面编写用例
2. **提升效率** — 一次配置，多项目复用
3. **保障质量** — 复杂场景用代码扩展，不妥协
4. **可持续** — 框架稳定，用例可积累，知识可传承

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| 配置驱动 | 元素定位、测试数据、项目配置均以文件/数据库存储，不硬编码 |
| 分层隔离 | 业务逻辑、元素定位、执行引擎三层分离 |
| 渐进复杂 | 简单用例YAML/Excel，复杂场景Python扩展 |
| 多端支持 | Web UI + 命令行 + API调用 |

## 四、新增功能需求分析

### 4.1 功能优先级

| 优先级 | 功能模块 | 建议理由 |
|--------|----------|---------|
| P1 | 接口检测 | 复用现有API客户端，改动小，收益快 |
| P1 | 性能检测 | 与SEO检测共用Playwright引擎，可并行开发 |
| P2 | 网站SEO检测 | 依赖Playwright渲染，SEO评分模型需与用户对齐 |
| P2 | 代理 | 被多个模块依赖（SEO/爬虫），建议早做 |
| P3 | 黑白盒测试 | 涉及架构调整，白盒安全风险需审慎设计 |
| P3 | 爬虫巡检 | 依赖爬虫本身成熟度，建议后置 |

### 4.2 各功能详细需求

#### 4.2.1 网站SEO检测

**背景：** 对目标网站进行SEO健康度检测，识别页面SEO问题，输出可量化的SEO评分报告。

**功能点：**
| 功能 | 描述 |
|------|------|
| 页面抓取 | 深度抓取指定URL，提取页面内容（支持SPA渲染） |
| Meta检测 | title/description/keywords/og:tags/canonical/hreflang |
| 内容分析 | H1-H6标签结构、图片alt、文字编码、重复内容 |
| 链接检测 | 死链检测、内外链分析、robots.txt可访问性 |
| 技术SEO | sitemap.xml、面包屑、结构化数据(JSON-LD)、AMP |
| 移动适配 | viewport配置、移动端可用性 |
| SEO评分 | 量化评分（Critical/Warning/Info分级） |
| 报告导出 | HTML报告 + JSON，支持飞书/钉钉通知 |
| 定时巡检 | Cron配置，支持与历史结果对比 |

**非功能需求：**
- 单页面检测≤5s（SPA≤15s）
- 并发检测≥10线程
- 死链超时≤10s/URL

---

#### 4.2.2 性能检测

**背景：** 采集Core Web Vitals（LCP/FID/CLS）及资源加载时序，定位性能瓶颈。

**功能点：**
| 功能 | 描述 |
|------|------|
| 页面性能采集 | Playwright + Performance API，获取FCP/LCP/FID/CLS/TTFB |
| 资源瀑布图 | 捕获JS/CSS/IMG/FONT的请求/响应时间 |
| 关键指标评分 | Lighthouse风格Performance Score |
| 慢页面定位 | TTFB慢、渲染阻塞、大资源问题 |
| 视频录制 | 可选：录制页面加载生成HAR |
| 终端模拟 | 移动端（375×667）/桌面端（1920×1080） |
| 基线对比 | 与历史数据对比，性能退化告警 |
| 报告导出 | HTML瀑布图 + JSON，支持通知 |

**非功能需求：**
- 单页面≤20s（含渲染等待）
- 移动/桌面双端支持
- 网络超时10s阈值

---

#### 4.2.3 接口检测

**背景：** 对API接口进行功能正确性、响应时间、可用性检测，支持参数化用例。

**功能点：**
| 功能 | 描述 |
|------|------|
| 请求构造 | GET/POST/PUT/DELETE/PATCH；Headers/Query/Body/Auth |
| 参数化 | 变量替换 `${var}`，YAML/Excel/CSV数据源 |
| 断言验证 | 状态码、响应时间、JSONPath/XPath提取、内容匹配 |
| 关联参数 | 提取上一请求响应字段，注入下一请求 |
| 环境隔离 | 多环境配置（dev/staging/prod） |
| 用例编排 | 用例依赖链（DAG顺序执行） |
| 并发检测 | 单接口多并发摸高，检测线程安全 |
| 报告导出 | 详细请求/响应日志，支持通知 |

**非功能需求：**
- 响应时间精度≤10ms
- 支持≥100并发请求
- 支持REST/GraphQL/WebSocket

---

#### 4.2.4 黑白盒测试

**背景：** 黑盒（功能流程）+ 白盒（代码覆盖率），统一管理两种测试模式。

**功能点：**
**黑盒部分：**
| 功能 | 描述 |
|------|------|
| 业务流程编排 | 串联多个UI操作为业务场景 |
| 数据驱动用例 | YAML/Excel定义步骤和预期 |
| 截图/录屏 | 失败自动截图；可选全程录屏 |
| 断言库 | 页面元素/URL/数据库/API断言 |
| 跨浏览器 | Chromium/Firefox/WebKit |

**白盒部分：**
| 功能 | 描述 |
|------|------|
| 单元测试接入 | 接入pytest，支持Python单元测试 |
| 覆盖率采集 | 接入coverage.py，生成line/branch覆盖率 |
| 代码扫描 | 静态分析（语法错误、未处理异常） |
| 报告合并 | UI测试+单元测试+覆盖率统一报告 |

**非功能需求：**
- 浏览器启动≤3s/实例
- 用例隔离（独立浏览器上下文）
- Python 3.8+兼容

---

#### 4.2.5 代理

**背景：** 提供代理池管理，支持HTTP/HTTPS/SOCKS5，用于请求分发和IP封禁规避。

**功能点：**
| 功能 | 描述 |
|------|------|
| 代理池管理 | 添加/删除/启用/禁用；健康检查（定期ping） |
| 代理协议 | HTTP/HTTPS/SOCKS5，支持认证（user:pass） |
| 代理调度 | 轮询/随机/失败切换策略；使用计数统计 |
| 代理来源 | 手动录入 + 代理服务商API接入 |
| 用量监控 | 可用率、平均响应时间、累计使用次数 |
| 本地代理 | 提供本地HTTP代理服务，汇总请求记录日志 |
| 通知 | 可用率<阈值自动告警 |

**非功能需求：**
- 代理请求延迟增加≤50ms
- 健康检查频率可配置（默认5min）
- 代理池至少保留3个可用代理

---

#### 4.2.6 爬虫抓取功能日常巡检

**背景：** 对爬虫脚本进行健康度巡检，确保爬虫服务稳定，发现异常及时告警。

**功能点：**
| 功能 | 描述 |
|------|------|
| 爬虫注册 | 注册已有爬虫脚本，配置巡检策略 |
| 存活检测 | 定时触发，验证进程存在+无报错 |
| 成功率检测 | 对比抓取量与预期基数，低于阈值告警 |
| 数据完整性 | 校验必填字段缺失告警 |
| 新鲜度检测 | 数据更新时间超时（>24h）告警 |
| 反爬检测 | 检测验证码/封禁，记录封禁IP |
| 执行日志 | 抓取耗时、数据量、错误信息 |
| 报告通知 | 巡检结果汇总，支持飞书/钉钉 |

**非功能需求：**
- 单次巡检≤60s
- 支持并发巡检多个爬虫
- 爬虫超时默认300s

| 风险 | 影响 | 对策 |
