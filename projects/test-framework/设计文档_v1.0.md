## 一、项目背景与目标

### 1.1 背景

AI时代开发周期缩短，测试需要：
- 用例编写门槛低（不懂代码也能参与）
- 新项目适配快（配置驱动，代码复用）
- 复杂场景覆盖（不牺牲测试深度）

### 1.2 目标

1. **降低门槛** — 非技术人员可通过Excel/Web界面编写用例
2. **提升效率** — 一次配置，多项目复用
3. **保障质量** — 复杂场景用代码扩展，不妥协
4. **可持续** — 框架稳定，用例可积累，知识可传承

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| 配置驱动 | 元素定位、测试数据、项目配置均以文件/数据库存储，不硬编码 |
| 分层隔离 | 业务逻辑、元素定位、执行引擎三层分离 |
| 渐进复杂 | 简单用例YAML/Excel，复杂场景Python扩展 |
| 多端支持 | Web UI + 命令行 + API调用 |

### 4.4 L3: Python关键字

```python
# cases/dataify/keywords/dataify_keywords.py
from engine import keyword, page

@keyword("登录dataify")
def login_dataify(username: str, password: str):
    """封装Dataify登录流程"""
    page.navigate("/dashboard/login/")
    page.click("login.tab_password")
    page.type("login.username", username)
    page.type("login.password", password)
    page.check("login.agree_terms")
    page.click("login.submit")
    page.wait_for_url("/dashboard/")

@keyword("创建任务")
def create_task(tool_name: str, params: dict):
    """创建爬虫任务"""
    page.navigate("/dashboard/tasks/create/")
    page.select("task.tool", tool_name)
    for key, value in params.items():
        page.type(f"task.param.{key}", value)
    page.click("task.submit")
    page.wait_for_alert()
```

### 4.5 L4: Python代码（复杂场景）

```python
# cases/dataify/test_complex.py
import pytest
from engine import api_client, page

class TestComplexScenarios:
    """复杂测试场景"""

    def test_任务并发创建_50个(self):
        """测试50个任务并发创建"""
        import concurrent.futures

        def create(i):
            return api_client.create_task(
                name=f"concurrent_task_{i}",
                tool="youtube_video_by-url",
                params={"url": f"https://youtube.com/watch?v={i}"}
            )

        with concurrent.futures.ThreadPoolExecutor(max_workers=10) as ex:
            results = list(ex.map(create, range(50)))

        passed = sum(1 for r in results if r.success)
        assert passed == 50, f"成功{passed}/50"

    def test_失败重试逻辑(self):
        """验证失败任务自动重试3次"""
        task_id = api_client.create_task(name="test_retry")
        result = api_client.wait_for_completion(task_id, timeout=300)

        assert result.attempts >= 3, "应该重试至少3次"
        assert result.final_status != "failed", "最终应该成功"
```

### 5.2 定位优先级规则

| 优先级 | 类型 | 示例 | 使用场景 |
|--------|------|------|---------|
| P1 | data-testid | `[data-testid='status-select']` | 最稳定，推荐优先 |
| P1 | id | `#username-input` | 唯一标识 |
| P2 | placeholder | `[placeholder*='搜索']` | 表单输入框 |
| P2 | text | `button:has-text('登录')` | 按钮/链接 |
| P3 | CSS路径 | `.login-form .submit` | 前两者找不到时 |
| P4 | XPath | `//input[@type='text']` | 最后手段 |

## 十三、风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| Locators维护成本高 | 页面改版后大量失效 | AI自动检测失效 + 版本管理 |
| 用例质量参差不齐 | 测不出问题 | 用例评审流程 + 代码审查 |
| 执行不稳定(Flaky) | 结果不可信 | 自动重试 + Flaky检测 + 隔离 |
| 数据污染 | 互相干扰 | 数据隔离策略 + 环境清理 |
| 团队推广难 | 没人用 | 渐进培训 + 激励措施 |
